顶点是什么

在3D图形的世界里，所有的物体都是由顶点构成的。无论是一个简单的立方体，还是一个复杂的游戏角色，本质上都是一堆顶点的集合。

顶点：3D世界的"原子"

想象你用乐高积木搭建一个城堡：

  • 每一块乐高积木就像一个三角形
  • 积木的每个角就像一个顶点
  • 整个城堡就是由无数个角（顶点）连接而成

在3D图形中，顶点就是最基本的构建单位，就像原子是物质的基本单位一样。

顶点不只是位置

很多人以为顶点就是一个3D坐标点，但实际上，一个顶点可以携带很多信息：

// 一个完整的顶点数据结构
const vertex = {
  // 位置 - 这个点在3D空间中的坐标
  position: { x: 1.0, y: 2.0, z: 3.0 },
  
  // 法线 - 这个点的表面朝向（用于光照计算）
  normal: { x: 0.0, y: 1.0, z: 0.0 },
  
  // 纹理坐标 - 这个点对应纹理图片的哪个位置
  texCoord: { u: 0.5, v: 0.5 },
  
  // 颜色 - 这个点的颜色（可选）
  color: { r: 1.0, g: 0.0, b: 0.0, a: 1.0 },
  
  // 骨骼权重 - 用于骨骼动画（可选）
  boneWeights: [0.5, 0.3, 0.2, 0.0],
  boneIndices: [0, 1, 2, 0]
};

为什么用三角形？

你可能会问：为什么3D模型都是用三角形，而不是正方形或其他形状？

原因很简单：

  1. 三角形永远是平的：三个点确定一个平面，不会弯曲
  2. 任何形状都能分解成三角形：正方形=2个三角形，五边形=3个三角形
  3. GPU专门优化了三角形处理：硬件层面的支持

顶点是如何连接的？

顶点本身只是一些点，需要用索引来告诉GPU如何把它们连成三角形：

// 一个正方形的顶点
const vertices = [
  { position: [-1, -1, 0] },  // 顶点0：左下
  { position: [ 1, -1, 0] },  // 顶点1：右下
  { position: [ 1,  1, 0] },  // 顶点2：右上
  { position: [-1,  1, 0] },  // 顶点3：左上
];

// 索引告诉GPU如何连接顶点
const indices = [
  0, 1, 2,  // 第一个三角形：顶点0-1-2
  0, 2, 3,  // 第二个三角形：顶点0-2-3
];

// 这样4个顶点就组成了2个三角形，拼成一个正方形

顶点数量与模型精度

顶点越多，模型就越精细：

  • 低模（Low Poly）：几百到几千个顶点，棱角分明
  • 中模：几千到几万个顶点，比较平滑
  • 高模（High Poly）：几十万甚至上百万顶点，非常精细

就像用马赛克拼图，格子越小（顶点越多），图像就越清晰。

顶点在渲染流水线中的旅程

顶点数据 → 顶点着色器 → 变换后的顶点 → 组装成三角形 → 光栅化 → 像素

每个顶点都会经过顶点着色器的处理，在那里可以：

  • 变换位置（移动、旋转、缩放）
  • 计算光照
  • 传递数据给片元着色器

小结

在这一章中，我们学到了：

  1. 顶点是3D模型的基本构建单位
  2. 顶点不只包含位置，还可以包含法线、纹理坐标、颜色等信息
  3. 三角形是3D图形的基本图元，由三个顶点组成
  4. 顶点数量决定了模型的精细程度
